x402-facilitator-go

This is a deployed instance of the x402-facilitator-go service brought to you by Raid Guild.

You can use this service to verify and settle payments for x402 transactions.

API Key

Depending on the configuration, you may need to provide an API key to access the API endpoints.

If this instance requires an API key, you will need to include the API key in the X-API-Key header.

API Endpoints

  • /api/verify (may require API key)
  • /api/settle (may require API key)
  • /api/supported (no API key required)

Verify Examples

An example request to the /api/verify endpoint (with API key):

curl -X POST api/verify \ -H "X-API-Key: your-api-key" \ -H "Content-Type: application/json" \ -d '{"x402Version": 1, "paymentPayload": {...}, "paymentRequirements": {...}}'

An example request to the /api/verify endpoint (without API key):

curl -X POST api/verify \ -H "Content-Type: application/json" \ -d '{"x402Version": 1, "paymentPayload": {...}, "paymentRequirements": {...}}'

An example response from the /api/verify endpoint (200 OK - valid):

{ "isValid": true, "payer": "0x..." }

An example response from the /api/verify endpoint (200 OK - invalid):

{ "isValid": false, "invalidReason": "invalid_authorization_signature" }

An example response from the /api/verify endpoint (401 Unauthorized):

unauthorized

An example response from the /api/verify endpoint (400 Bad Request):

invalid character 'x' looking for beginning of value

An example response from the /api/verify endpoint (500 Internal Server Error):

failed to get token balance

curl -X POST api/settle \ -H "Content-Type: application/json" \ -d '{"x402Version": 1, "paymentPayload": {...}, "paymentRequirements": {...}}'
